1. Общее описание

Необходимо разработать Telegram-бота, который представляет собой интерактивный текстовый квест. Пользователь ("игрок") должен пройти регистрацию и двигаться по сюжету, делая выбор в ключевых точках повествования. Прогресс пользователя (текущее местоположение в квесте, инвентарь, характеристики) должен сохраняться между сеансами общения с ботом.

2. Основные функциональные требования

2.1. Система команд
Бот должен обрабатывать и реагировать на следующие стандартные команды:

/start — начинает взаимодействие с ботом. Если пользователь новый, ему предлагается зарегистрироваться. Если пользователь уже зарегистрирован, бот должен "вспомнить" его и продолжить с места сохранения.
/help — выводит список всех доступных команд и краткое описание игры.
/reset — полностью сбрасывает прогресс текущего пользователя (удаляет все его данные) и начинает квест заново.
/status (или /stats) — показывает текущее состояние игрока: имя, текущую локацию, количество очков, инвентарь и т.д.

2.2. Система регистрации и хранения прогресса

При первом использовании команды /start бот должен запросить у пользователя его имя (или никнейм) для регистрации.
Все данные о пользователе должны сохраняться в структурированном файле. Можно использовать формат JSON (рекомендуется).

Ключевые данные для сохранения:

user_id — уникальный идентификатор пользователя в Telegram.
user_name — имя, выбранное пользователем.
current_scene — идентификатор текущей локации/сцены в квесте.
inventory — список предметов, которые есть у игрока.
points — количество очков (или здоровья, репутации).

2.3. Логика квеста

Квест должен состоять как минимум из трех взаимосвязанных локаций (сцен).
Каждая сцена должна содержать:

Текстовое описание обстановки.
Вопрос или проблему.
Варианты действий (как минимум 2-3), которые пользователь может выбрать. Для выбора действий используйте ReplyKeyboardMarkup (как в боте-конвертере валют).
Выбор пользователя должен влиять на развитие сюжета (приводить к разным сценам) и/или на его характеристики (добавлять предмет в инвентарь, изменять количество очков).


Пример простого сюжета:

Сцена 1 (Старт): "Вы стоите на развилке двух дорог. Куда пойдете?" -> Варианты: ["Налево, в лес", "Направо, к реке"].
Сцена 2 (Лес): "В лесу вы нашли сундук. Открыть его?" -> Варианты: ["Да", "Нет, пойти дальше"].
Сцена 3 (Река): "Через реку ведет хлипкий мост. Перейти?" -> Варианты: ["Перейти", "Поискать другой путь"].
